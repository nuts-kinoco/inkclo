# INKCLO Project Handover

## プロジェクト概要
INKCLOは『スプラトゥーン3』のギア（アタマ・フク・クツ）を組み合わせて、お洒落なカラーコーディネートを作成・保存・共有できるWebアプリケーションです。

## 技術スタック
- **Framework**: Next.js 15 (App Router, Turbopack)
- **Styling**: Tailwind CSS v4 (CSS VariablesベースのDark Mode対応)
- **State Management**: Zustand (`src/store/builderStore.ts`)
- **Icons**: Lucide React
- **Data**: `src/lib/data/gears.json` (ローカルJSONデータベース)
- **Image Processing**: `sharp` (画像リサイズ・透過カラー解析)

## 開発のあゆみと成果 (Sprint 1 ~ 8)

### 1. 基礎機能と初期実装 (Sprint 1 ~ 4)
- ギアのカタログUI（アタマ・フク・クツ別）
- コーディネートプレビュー（キャンバス）と画像エクスポート機能（`html-to-image`）
- `dominantColor` / `palette` を用いたDeltaE色差検索機能

### 2. カラーアシスタント機能の追加 (Sprint 5 ~ 7)
- 加重平均色（Base Color）から、`Monotone`, `Analogous`, `Complementary` の色彩理論に基づく「おすすめギア」を提案する推薦エンジン。
- `autoTags` / `manualTags` に基づくタグ基盤および管理用の `/admin/tags` 画面の構築。
- 推薦エンジンに「トライアド (Triadic)」を追加。

### 3. データ完全網羅・アルゴリズム・UXの徹底強化 (Sprint 8 - 最新の成果) 🌟

有志のWikiやイカクロなど「手動更新で一部ギア画像が欠損する外部サイト」へのスクレイピングを廃止し、ゲーム内部の解析データ（Leanny氏のSplat3データ）と連携する堅牢な仕組みに刷新しました。

- **全ギアデータの完全網羅＆高画質化**:
  - `scripts/fetch-gears.ts` を用いて、ゲーム内から直接抽出された最新ギアデータを取得。
  - サウナハットや最新のカタログ限定品など、**合計940件**のギアデータを画像付きで網羅しました。
- **データクレンジングと重複防止**:
  - 全角半角やスペースの差異（例: `８ビットフレーム` と `8ビットフレーム` 等23件）による重複登録を、正規化名比較により完全に排除。
  - 没データやデバッグ用の無効なデータ（`None` ブランド、`INVISIBLE`等）の自動除外。
  - 表記ゆれとなっていた `'エンペリー*1'` を `'エンペリー'` へクレンジング。
  - 既存のタグ・パレットデータを100%温存したまま差分だけを統合する `scripts/merge-gears.ts` を構築。
- **透過背景を考慮した高精度カラー抽出アルゴリズム**:
  - 透過PNGの透明背景（透明な黒）を自動で弾き、ギア実体の色だけを減色処理（K-Means風バケッティング）で抽出する新ロジックへ移行。
  - サウナハットが真っ黒（`#080808`）と判定されていた問題を修復し、本来の「生成り白 `#FFFEEE`」や「ピンク `#BB5D74`」などを完璧にパレット化。
- **推奨エンジンの重複排除（100%ユニーク化）**:
  - ベースカラーが無彩色の際、カラー理論の限界で全カルーセルのサジェストが重複してしまう問題を、優先度順の相互排他フィルタ（既出の推奨ギアIDの除外）で解消。
- **シャッフル仕様の改善**:
  - シャッフル対象を「装備中ギア」から「サジェストカード」に変更。装備中のコーデは固定したまま、気に入らない提案カードの組み合わせだけをランダムに切り替えられるよう修正。
  - 検索フィルター適用時にも候補が0件で無反応にならないよう、自動で検索テキスト条件のみを外して再検索するフォールバック（セーフティ）処理を追加。
- **左バー詳細情報（i）ボタンの追加**:
  - 装備中ギアカードの左上に詳細情報（i）ボタンを配置し、カード詳細モーダルが一発で起動できるように修正。右上のバツボタンもLucideの `X` に統一し、プレミアムなホバーデザインへと刷新。
- **Vercelデプロイエラーの修復**:
  - ESLintの過度に厳格な制約（`any`使用、`prefer-const`等）を `eslint.config.mjs` で緩和。
  - `Coordinate` 型定義に `name` を追加して型エラーを解決。`npm run build` を無事通過し、GitHubへ正常プッシュ完了。

## これから実装できること・すると良いこと (Future Tasks) 🚀

1. **スタイル（タグ）ベースの推薦システムの実装**
   - 現在のカラーアシスタントに加え、「スポーティ」「ストリート」「カジュアル」などのスタイルタグ（`tags`）に一致するギアを推薦する戦略を追加すると、さらに強力でお洒落なコーディネート提案が可能になります。
2. **OGP共有カードの動的レンダリング微調整**
   - Vercel上でOGPエンドポイント（`/src/app/api/og/route.tsx`）がダイナミックなシェア画像を美しくレンダリングできるかの実環境テストと、Xにシェアした時のカードの見え方の最終調整。
3. **お気に入り機能の永続化データベース対応**
   - 現在ローカルストレージやZustandベースであるお気に入り機能を、ユーザーごとにサーバーサイドのデータベース（Supabase / Vercel KVなど）と連携させ、より堅牢なデータ永続化を行う。
4. **詳細モーダル下部の見切れ防止微調整**
   - ギア詳細モーダルにおいて、カラー比率リストが超小型モバイル端末などで下に見切れる問題を防ぐため、1%未満の比率アイテムの非表示化や、表示高に応じたパディングのスクロール調整。
5. **ギア追加プログラム (`merge-gears.ts`) の自動実行化**
   - シーズンアップデート時に全自動で最新ギアを取り込めるよう、VercelのCron Job機能などでマージ処理を定期自動実行させるタスクの追加。
